<span data-ttu-id="c7fcf-177">这有点棘手。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-177">This is a little more tricky.</span></span> <span data-ttu-id="c7fcf-178">通过使用全息的 "旋转" 按钮，您可以获得更丰富的体验。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-178">Having holographic “rotate” buttons to tap makes for a terrible experience.</span></span> <span data-ttu-id="c7fcf-179">没有要利用的物理控件，因此该行为必须从表示 lander 的对象或 lander 本身的操作来处理。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-179">There isn’t a physical control to leverage, so the behavior must come from manipulation of an object representing the lander, or with the lander itself.</span></span> <span data-ttu-id="c7fcf-180">我们提供了一种使用点击和-拖动的方法，使用户能够有效地 "推送并拉取" 其所需的方向。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-180">We came up with a method using tap-and-drag which enables a user to effectively “push and pull” it in the direction they want it to face.</span></span> <span data-ttu-id="c7fcf-181">用户点击并保存时，启动笔势的空间中的点就成为旋转的原点。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-181">Any time a user taps and holds, the point in space where the gesture was initiated becomes the origin for rotation.</span></span> <span data-ttu-id="c7fcf-182">从原点拖动会将手平移的增量转换 (X、Y、Z) ，并将其应用于 lander 旋转值的增量。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-182">Dragging from the origin converts the delta of the hand's translation (X,Y,Z) and applies it to the delta of the lander's rotation values.</span></span> <span data-ttu-id="c7fcf-183">更简单的一点是， *向上拖动 <-> 右，向上 <-> 向下，向前 < 中的 > 返回空格，并相应地旋转*。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-183">Or more simply, *dragging left <-> right, up <-> down, forward <-> back in spaces rotates the ship accordingly*.</span></span>

<span data-ttu-id="c7fcf-184">由于 HoloLens 可以跟踪两次，因此，在主旨是由左侧控制时，可以向右手分配旋转。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-184">Since the HoloLens can track two hands, rotation can be assigned to the right hand while thrust is controlled by the left.</span></span> <span data-ttu-id="c7fcf-185">Finesse 是在此游戏中取得成功的驱动因素。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-185">Finesse is the driving factor for success in this game.</span></span> <span data-ttu-id="c7fcf-186">这些交互的 *外观* 是绝对最高的优先级。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-186">The *feel* of these interactions is the absolute highest priority.</span></span> <span data-ttu-id="c7fcf-187">尤其是在 tactile 浸入式的上下文中。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-187">Especially in context of tactile immersion.</span></span> <span data-ttu-id="c7fcf-188">如果发货反应太快，反应速度太慢，而速度太慢，则会要求用户在 awkwardly 长时间内 "推送并请求"。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-188">A ship that reacts too quickly would be unnecessarily difficult to steer, while one too slow would require the user to “push and pull” on the ship for an awkwardly long amount of time.</span></span>

### <a name="adding-input-for-game-controllers"></a><span data-ttu-id="c7fcf-189">添加游戏控制器的输入</span><span class="sxs-lookup"><span data-stu-id="c7fcf-189">Adding input for game controllers</span></span>

<span data-ttu-id="c7fcf-190">尽管 HoloLens 上的手势提供了 novel 的精细控制方法，但仍有特定的 "true" tactile 反馈需要从模拟控件获取。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-190">While hand gestures on the HoloLens provide a novel method of fine-grain control, there is still a certain lack of 'true' tactile feedback that you get from analog controls.</span></span> <span data-ttu-id="c7fcf-191">通过连接 Xbox 游戏控制器，可以使 physicality 的这一意义，同时利用控制杆来保持精细的控制。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-191">Connecting an Xbox game controller allows to bring back this sense of physicality while leveraging the control sticks to retain fine-grain control.</span></span>

<span data-ttu-id="c7fcf-192">可以通过多种方式将相对直接的控件方案应用于 Xbox 控制器。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-192">There are multiple ways to apply the relatively straight-forward control scheme to the Xbox controller.</span></span> <span data-ttu-id="c7fcf-193">由于我们尝试尽可能接近原始拱廊类设置，因此最好将 **主旨是** 映射到触发器按钮。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-193">Since we're trying to stay as close to the original arcade set up as possible, **Thrust** maps best to the trigger button.</span></span> <span data-ttu-id="c7fcf-194">这些按钮是模拟控件，这意味着它们的 *开启和关闭* 状态大于简单状态，它们实际上会响应施加的压力。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-194">These buttons are analog controls, meaning they have more than simple *on and off* states, they actually respond to the degree of pressure put on them.</span></span> <span data-ttu-id="c7fcf-195">这为我们提供了类似于 **主旨是杠杆** 的结构。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-195">This gives us a similar construct as the **thrust lever**.</span></span> <span data-ttu-id="c7fcf-196">与原始游戏和手手势不同，此控制会在用户停止对触发器的压力后，就会主旨是。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-196">Unlike the original game and the hand gesture, this control will cut the ship's thrust once a user stops putting pressure on the trigger.</span></span> <span data-ttu-id="c7fcf-197">它仍为用户提供与原始拱廊类游戏相同的 finesse。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-197">It still gives the user the same degree of finesse as the original arcade game did.</span></span>

<span data-ttu-id="c7fcf-198">![左操纵杆映射到偏航并滚动，将右操纵杆映射到螺距并滚动](images/thumbsticksidebyside.gif)</span><span class="sxs-lookup"><span data-stu-id="c7fcf-198">![Left thumbstick is mapped to Yaw and Roll, Right thumbstick is mapped to Pitch and Roll](images/thumbsticksidebyside.gif)</span></span><br>
<span data-ttu-id="c7fcf-199">*左操纵杆映射到偏航并滚动向右操纵杆映射到间距和滚动*</span><span class="sxs-lookup"><span data-stu-id="c7fcf-199">*Left thumbstick is mapped to yaw and roll; right thumbstick is mapped to pitch and roll*</span></span>

<span data-ttu-id="c7fcf-200">双重 thumbsticks 自然会为控制运输旋转提供控制。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-200">The dual thumbsticks naturally lend themselves to controlling ship rotation.</span></span> <span data-ttu-id="c7fcf-201">遗憾的是，发货可以旋转3个轴，两个 thumbsticks 支持两个轴。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-201">Unfortunately, there are 3 axes on which the ship can rotate and two thumbsticks which both support two axes.</span></span> <span data-ttu-id="c7fcf-202">这种不匹配意味着一个操纵杆控制一个轴;或者 thumbsticks 的轴重叠。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-202">This mismatch means either one thumbstick controls one axis; or there is overlap of axes for the thumbsticks.</span></span> <span data-ttu-id="c7fcf-203">以前的解决方案最终会 "损坏"，因为 thumbsticks 原本会混合其本地 X 和 Y 值。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-203">The former solution ended up feeling "broken" since thumbsticks inherently blend their local X and Y values.</span></span> <span data-ttu-id="c7fcf-204">后一种解决方案需要进行一些测试，以发现哪些冗余轴感觉最自然。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-204">The latter solution required some testing to find which redundant axes feel the most natural.</span></span> <span data-ttu-id="c7fcf-205">最后一个示例使用 *偏航* 并 *滚动* 使用左侧操纵杆 (Y 和 x 轴) *，并为* 右操纵杆 *)  (Z 和 x* 轴。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-205">The final sample uses *yaw* and *roll* (Y and X axes) for the left thumbstick, and *pitch* and *roll* (Z and X axes) for the right thumbstick.</span></span> <span data-ttu-id="c7fcf-206">这种感觉最自然的 *做法是，* 在 *偏航* 和 *音调* 上，自然会使其不受影响。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-206">This felt the most natural as *roll* seems to independently pair well with *yaw* and *pitch*.</span></span> <span data-ttu-id="c7fcf-207">作为一条注释，同时使用两个 thumbsticks 进行 *滚动* 还会使旋转值翻倍;让 lander 执行循环非常有趣。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-207">As a side note, using both thumbsticks for *roll* also happens to double the rotation value; it's pretty fun to have the lander do loops.</span></span>

<span data-ttu-id="c7fcf-208">此示例应用演示了如何通过 Windows Mixed Reality 的可扩展输入情态，空间识别和 tactile 浸入式可以显著改变体验。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-208">This sample app demonstrates how spatial recognition and tactile immersion can significantly change an experience thanks to Windows Mixed Reality's extensible input modalities.</span></span> <span data-ttu-id="c7fcf-209">虽然农历 Lander 可能接近40年，但使用这种小小的小小的概念却会永远不会出现。</span><span class="sxs-lookup"><span data-stu-id="c7fcf-209">While Lunar Lander may be nearing 40 years in age, the concepts exposed with that little octagon-with-legs will live on forever.</span></span> <span data-ttu-id="c7fcf-210">在应该构想将来，为什么不会看到过去？</span><span class="sxs-lookup"><span data-stu-id="c7fcf-210">When imagining the future, why not look at the past?</span></span>
